## For Developers

Color styling is located in the `./v2/src/styles/_tokens/_colors.scss` and is part of `mise.scss`.

### Using the Color Palette

Colors can be used with the `mise.color()` function. This custom SASS function takes a `category` and optional `hue`, and `variant` arguments. All colors should be implemented with `mise.color()`, **no hex codes, rgba, sass transforms, etcâ€¦** should be used directly in component styling. 

> **Note:** All colors have the `variant`: `base`, the default color. `hues` vary by `category`, see the [color palette](#color-palette) below for available options.

#### SASS | *.scss

```
@use "./src/styles/mise.scss"

// Example: mise.color(category, hue, variant)

.example {
  background-color: mise.color(primary, mid, base);
  border-color: mise.color(primary, dark, base);
  color: mise.color(netural, light, tint);

  &:hover {
    background-color: mise.color(primary, mid, shade);
    border-color: mise.color(primary, dark, shade);
  }
}
```

## <a name="color-palette" />Color Palette

The following colors are **the only colors that can be used** to style Mise components.

{Object.keys(Colors).map((category, i) => {
  return (
    <>
      {i === 0 ? "" : <hr />}
      <h3><a name={category} />{getTokenTitle(category)}</h3>
      <ColorPalette>
        {Object.keys(Colors[category]).map((hue) => {
          return <ColorItem title={getTokenTitle(hue)} subtitle={hue} colors={Colors[category][hue]} />
        })}
      </ColorPalette>
    </>
  )
})}
